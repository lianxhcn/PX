## 第〇章 · 新手快速上手（强烈建议先读）

### 0.1 上课前需要准备哪些软件？（标准清单）

**出现频率**：★★★★★

无论你最终用 Claude Code、OpenCode 还是 Codex，**安装教程里除 OpenCode 以外的所有配套软件都需要装好**。完整清单：

| 软件 | 作用 | 必装 |
| --- | --- | --- |
| **VS Code** | 工作台 / 代码与文档编辑器 | ✅ |
| **Node.js（含 npm）** | 安装 Claude Code / OpenCode 的运行环境 | ✅ |
| **Miniconda（或已装 Anaconda）** | Python 环境管理 | ✅ |
| **Git** | 版本控制、拉取课程仓库 | ✅ |
| **Pandoc** | 文档格式转换 | ✅ |
| **Claude Code CLI**（终端版） | 课堂主力 AI 工具 | ✅（推荐） |
| **cc-switch** | 管理/切换模型 API（接 DeepSeek 等） | ✅（推荐） |
| **约 15–20 个实用 Skills** | 增强 AI 的科研能力 | ✅ |
| OpenCode | 备用 AI 工具，CC 出问题时用来"自我修复" | 可选 |
| TeXLive 或 MiKTeX | 编译 LaTeX / 生成 PDF（写综述时用） | 按需 |

> **课前最该掌握的资料**：「附录 A：Claude Code 基础」请课前掌握并练习；A1–A4 是授课讲义。其他附件是补充技术资料，需要时再查或直接让 AI 读。
> 先导课配套推文：《从 OpenCode 到 VS Code 到 Skills：一次配齐整套 AI 生产力工具》。

### 0.2 这些工具分别是什么、怎么选？（一次说清）

- **Claude Code（简称 CC）**：Anthropic 官方出品的"终端 AI 代理"，默认锁定 Claude 模型，课堂演示以它为主。**它是"手和脚"**（负责读文件、跑命令、改代码）。
- **OpenCode（简称 OC）**：第三方开源的 AI 工作平台，原生支持接入多种模型。本身不是模型，是"平台"。可作为 CC 的备用与诊断修复工具。
- **模型（DeepSeek / 智谱 GLM / Kimi / Claude / GPT 等）**：才是"大脑"，负责思考与生成内容。CC/OC 只是外壳，真正干活的是模型。
- **cc-switch**：配置管理 + 一键切换工具，统一管理不同模型的 API Key / BaseURL / 模型名，让 CC 轻松接入国内模型，**避免手动配置环境变量**。它不是 AI 工具。
- **Skills**：教 AI"怎么把一件事做好"的方法包（工作流程 + 提示词模板），本质多是说明文档。
- **Plugins**：Claude Code 专属，本质是"打包好的 agents 配置 + skills 文件"，是一套自动化工作流。
- **MCP**：让 AI 连接外部工具/数据库/接口的"接口标准"（如连 Zotero、Stata、浏览器、联网搜索）。

> **一句话关系**：cc-switch 管 API ＋ CC/OC 当外壳 ＋ 模型当大脑 ＋ Skills 教方法 ＋ MCP 接外部工具。

### 0.3 国内学员推荐路线（无需稳定梯子）

**出现频率**：★★★★★

最省心、最稳的国内方案：

1. 装好 0.1 清单里的全部基础软件。
2. 用国内镜像安装 Claude Code：
   ```
   npm install -g @anthropic-ai/claude-code --registry=https://registry.npmmirror.com
   ```
3. 安装 **cc-switch**（下载：<https://github.com/farion1231/cc-switch/releases>，视频教程：<https://www.bilibili.com/video/BV1L7c7zZEAt/>）。
4. 到 <https://platform.deepseek.com/> 注册并申请 API Key，**先充约 100 元即可**（DeepSeek 余额很耐用）。
5. 在 cc-switch 图形界面里添加 DeepSeek 供应商、填入 Key、把模型设为 `deepseek-v4-pro`，点"启用"。
6. 在终端进入任意工作目录，输入 `claude` 说"你好"测试。

> 这样既不需要国外银行卡，也基本不需要稳定梯子（DeepSeek 是国内服务），还能直接跟做课堂演示。预算充裕、追求最佳效果者可考虑 Claude 官方订阅或 Claude Opus，但门槛和封号风险更高（见[第九章](#第九章--模型选择与对比)、[第十二章](#第十二章--课程相关资料回放设备播放器测试订阅支付)）。

### 0.4 如何确认环境配置成功？（通用自检清单）

**出现频率**：★★★★★

逐条在终端验证，能显示版本号或进入对话框即为成功：

| 检查项 | 命令 | 成功标志 |
| --- | --- | --- |
| Node / npm | `node -v` / `npm -v` | 显示版本号 |
| Git | `git --version` | 显示版本号 |
| Conda | `conda --version` | 显示版本号（在 Anaconda Prompt 里一定可以） |
| Claude Code | `claude --version` 或输入 `claude` 回车 | 显示版本号 / 出现对话框 |
| OpenCode | `opencode --version` 或输入 `opencode` | 显示版本号 / 进入文字界面 |
| Skills | 在 CC 中输入 `/skills` | 出现技能列表 |
| VS Code 插件 | 扩展面板（<kbd>Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>X</kbd>） | 已安装项显示绿色对勾 |

- 如果只显示路径或"没反应"，**先多按一次回车**，或重新打开一个新终端再试（环境变量需重开终端才生效）。
- 能与工具正常一问一答，就说明配置成功。
- 截图很难靠人眼判断成败时，最快的办法是**把截图发给 DeepSeek 网页版，聊两个来回**就能确认。

> **AI 辅助提示词**：「我刚装完连享会课程要求的环境（Node、Git、conda、Claude Code、cc-switch、Skills）。这是我在终端逐条运行 `node -v`/`git --version`/`conda --version`/`claude --version`/`/skills` 的输出截图，请逐项判断是否安装成功，没成功的请给出修复命令。」

## 第一章 · 基础软件安装

### 1.1 VS Code

**Q1：VS Code 在哪里下载？已经装过还要重装吗？**（★）
- 官网下载：<https://code.visualstudio.com/>。
- **已装过不要删**，直接升级即可：打开 VS Code，顶部一般会有升级提示，或菜单 **帮助 → 检查更新**。VS Code 通常会自动更新。

**Q2：安装 Office Viewer / Git Graph / Marp 等插件失败怎么办？**
1. 完整退出 VS Code（任务管理器确认无残留进程）。
2. 右键 **以管理员身份运行** VS Code 再安装。
3. 仍失败：进入 `C:\Users\用户名\.vscode\extensions\`，删除 `.obsolete` 文件和对应插件残留文件夹后重试。

**Q3：在哪里以管理员身份运行 VS Code？**（★）
右键 VS Code 图标 → **以管理员身份运行**；或右键任务栏图标 → 再右键程序名 → 以管理员身份运行。

**Q4：VS Code 里怎么看 PPT？**
VS Code 不能直接读 PPT。把 PPT 转成 PDF 后在 VS Code 查看，或直接用 PowerPoint 打开。

**Q5：Marp 与 Quarto 插件冲突（Marp 按钮消失）**
`settings.json` 里写 `"*.md": "quarto"` 会把所有 md 强制关联到 quarto，导致 Marp 失效。改为只关联 qmd 即可共存：
```json
"files.associations": { "*.qmd": "quarto" }
```

**Q6：命令行 push 成功，但 VS Code 源代码管理器不显示远程仓库**
终端出现 `branch 'main' set up to track ...` 说明已连接成功。VS Code 界面不会立即刷新：暂存 → 提交 → 点同步/刷新按钮，或重启 VS Code。

**Q7：点击 "Publish to GitHub" 没反应**
多为未登录或授权失效。检查左下角账号是否已登录，退出后重新 **Sign in with GitHub** 完成浏览器授权，重启 VS Code，或 <kbd>Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>P</kbd> 搜索 "Git: Publish to GitHub"。网络不通时，先在 GitHub 网页建空仓库，再手动关联推送：
```bash
git remote add origin <仓库地址>
git push -u origin main
```

### 1.2 Node.js 与 npm

**Q1：装完 Node.js 点 finish 后电脑自动装很多东西，要一直等吗？**（★）
这是 Node.js 在自动安装可选的构建工具链（如 chocolatey / VS Build Tools），**对本课程不是必需**。可以等它装完，也可以关掉那个额外窗口，不影响 `node -v`/`npm -v` 的使用。装完在终端验证版本号即可。

**Q2：npm 报错 "Unexpected token '<'" / "is not valid JSON" / 一直转圈**（★★）
npm 下载源出错，拿到的是网页而非包信息。先切回官方源并清缓存：
```
npm config set registry https://registry.npmjs.org/
npm cache clean --force
```
连不上官方源时换网络/挂代理，或临时切国内镜像：`npm config set registry https://registry.npmmirror.com`。

**Q3：npm 全局命令目录怎么加入 PATH？（新版没有 `npm bin -g`）**
1. 确认 `C:\Users\<用户名>\AppData\Roaming\npm` 下有 `opencode.cmd` / `claude.cmd` 等文件。
2. 把该目录加入用户 PATH：**此电脑 → 属性 → 高级系统设置 → 环境变量 → 用户变量 Path → 新建 → 粘贴路径**。
3. 关闭所有 PowerShell 窗口重新打开。

**Q4：npm 网络超时连不上官方源**
换网络环境，或临时切国内镜像 `npm config set registry https://registry.npmmirror.com` 后重装。

### 1.3 Miniconda 与 Python 环境

**Q1：普通 PowerShell 里 `conda` 不识别（Anaconda Prompt 里却正常）**（★★）
> 这不代表没装好。普通 PowerShell 未初始化/未配 PATH 才不识别。

**标准修复步骤：**
1. 开始菜单打开 **Anaconda PowerShell Prompt**（或 Anaconda Prompt）。
2. 运行 `conda init powershell`。
3. 关闭所有终端，**重新打开一个新的普通 PowerShell**，应看到 `(base)` 提示符，`conda --version` 正常输出。
4. 仍不行：以管理员身份打开 Anaconda Prompt 重试，或让 OpenCode 帮忙："帮我在 PowerShell 初始化 conda 并把 base 设为默认环境自动加载"。

**Q2：`conda init powershell` 报错"无法将 conda 项识别为 cmdlet"**（★）
说明当前终端找不到 conda。请在 **Anaconda Prompt** 里执行该命令。若仍报错，先设置执行策略再重试：
```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```
输入 Y 确认后，以管理员身份打开 Anaconda Prompt 再 `conda init powershell`，完成后关闭所有窗口重开。

**Q3：`conda init powershell` 显示 "No action taken"，conda 还是不可用**
"No action taken" 表示初始化配置已写过，**不是失败**。关闭所有 PowerShell 和 VS Code 重新打开后测 `conda --version`。仍不识别就直接在 Anaconda Prompt 里用 conda，功能完全一样。

**Q4：Miniconda 安装时 "Add to PATH" 到底勾不勾？**
Anaconda 官方标注 "Not recommended"（不推荐勾选），以免与其他 Python 工具冲突。不勾也可用 `conda init powershell` 达到同等效果。**已装时没勾也无需重装**。

**Q5：Miniconda 安装包版本怎么选？已有 Python / Anaconda 还要装吗？**（★）
- 安装界面里带对钩的"Miniconda3 Windows 64-bit"类版本就是对的，本质同一个包。
- **已装 Anaconda 则无需再装 Miniconda**。
- 已有系统 Python 仍**建议装 Miniconda**：它能建独立虚拟环境、避免依赖冲突，且与系统 Python 共存、不会覆盖。

**Q6：安装路径报错 / "Python Not Found"**
- 安装路径**不能含中文**（用户名也尽量别用中文）。把安装位置改到 D 盘等非系统盘的纯英文路径。
- 运行中偶报 "python not found" 往往只是缺某个依赖包，AI 检测到后会在输出底部自动开始安装，属正常流程。

### 1.4 PowerShell 与 Windows Terminal（Windows）

**Q1：PowerShell 里 <kbd>Ctrl</kbd>+<kbd>V</kbd> 无法粘贴**（★）
Win10 旧版 PowerShell（5.1 及更早）默认不支持 <kbd>Ctrl</kbd>+<kbd>V</kbd>。两种解决：
- **推荐**：装 **Windows Terminal**（<https://apps.microsoft.com/detail/9n0dx20hk701>）替换旧版，之后可用 <kbd>Ctrl</kbd>+<kbd>V</kbd> 或 <kbd>Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>V</kbd>。
- 临时：右键标题栏 → 属性 → 选项 → 勾选"使用 Ctrl+Shift+C/V 作为复制/粘贴"。

**Q2：Windows Defender SmartScreen 拦截安装程序（"Windows protected your PC"）**
点 **更多信息（More info）→ 仍要运行（Run anyway）**。这是对未签名程序的常规提示，不是病毒。

**Q3：设置 `Set-ExecutionPolicy` 后终端没反应 / 报拼写错误**
- **没反应是正常的**，说明命令已成功执行，可继续下一步。
- 正确命令是 `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser`，注意 `RemoteSigned` 与 `-Scope` 之间**必须有空格**（常见错写成 "Remotesiged"）。

**Q4：Windows 需要装 Homebrew 吗？**
不需要。Homebrew 是 macOS 的命令行软件管理器，Windows 用 PowerShell + npm 即可。（PowerShell≈Mac 终端，npm≈Homebrew 那样的包管理器。）

### 1.5 Git

**Q1：`git config` 配置身份时显示 error**
多为终端混入了不可见字符。**手动逐字输入**（不要直接粘贴含全角符号的命令）：
```bash
git config --global user.name "yourname"
git config --global user.email "youremail"
git config --global --list   # 验证
```
姓名、邮箱仅用于提交记录，姓名用英文名、邮箱用常用邮箱即可。

**Q2：装 Git 时没勾选"加入 PATH"，要重装吗？**
不用。没勾只是没写入系统路径，并非没装好。先 `git --version` 测试，能显示版本号即已装成功；需要时手动把 Git 路径加进 PATH，或重跑安装包选 **Repair**，在 "Adjusting your PATH" 步骤选 "Git from the command line and also from 3rd-party software"。

**Q3：Git 安装包下载链接（GitHub）打不开**
这是 GitHub 网络不通，不是电脑故障。先 `git --version` 看是否已装；未装则换网络或用国内镜像下载 Git 安装包。

**Q4：怎么 fork 别人的仓库？**
在 GitHub 仓库页面点 **Fork** 即可。想让 AI 代劳，复制仓库网址告诉它："fork 这个链接 + 网址"。

### 1.6 Homebrew（macOS 专属）

**Q1：Homebrew 安装持续报错 / 连不上 raw.githubusercontent.com / portable-ruby 下载失败**（★）
主因是网络受限。按顺序尝试：
1. 先测连通性：`curl -I https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh`，通了再装。
2. 用国内镜像安装：
   ```bash
   /bin/bash -c "$(curl -fsSL https://gitee.com/ineo6/homebrew-install/raw/master/install.sh)"
   ```
3. 切换中科大镜像源后更新：
   ```bash
   export HOMEBREW_BREW_GIT_REMOTE="https://mirrors.ustc.edu.cn/brew.git"
   export HOMEBREW_CORE_GIT_REMOTE="https://mirrors.ustc.edu.cn/homebrew-core.git"
   export HOMEBREW_BOTTLE_DOMAIN="https://mirrors.ustc.edu.cn/homebrew-bottles"
   export HOMEBREW_API_DOMAIN="https://mirrors.ustc.edu.cn/homebrew-bottles/api"
   brew update-reset && brew update
   ```
4. 多行命令要**逐行复制、逐行回车**，不要一次性粘贴；`export` 前若有不可见字符，只从 `export` 开始选取。

**Q2：安装时没提示输入密码 / `brew doctor` 显示 warnings，是不是失败了？**（★）
- 命令里的横杠可能是中文全角横杠，让大模型**重新输出一遍命令**再复制即可。
- `brew doctor` 显示 warnings **是正常的**，说明 Homebrew 已能运行。
- 输密码时**屏幕不显示任何字符**（无星号、无小黑点），看起来像没反应——这是 macOS 的安全设计，放心输完按回车即可。

**Q3：Mac 终端提示 "command not found: claude"（或 brew 装的 node/code 找不到）**
brew 路径未写入 shell 配置。执行后重开终端：
```bash
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zshrc
eval "$(/opt/homebrew/bin/brew shellenv)"
```
- `code` 命令找不到：VS Code 内 <kbd>Cmd</kbd>+<kbd>Shift</kbd>+<kbd>P</kbd> 搜 "Shell Command: Install 'code' command in PATH"。
- `brew --prefix node` 能返回路径就说明 node 安装无问题。

**Q4：`brew install git node pandoc anomalyco/tap/opencode` 报错 / Mac M 系列 clone 失败**（★）
先把 Git 切到 HTTP/1.1 再装：
```bash
git config --global http.version HTTP/1.1
brew install anomalyco/tap/opencode
```
仍失败可用 `npm install -g opencode-ai` 替代 brew 安装 OpenCode；其余软件可走 Q1 的镜像方案。

**Q5：Mac 上 git、node 等装失败，是网络原因吗？**
是。切清华镜像源：
```bash
echo 'export HOMEBREW_API_DOMAIN="https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles/api"' >> ~/.zshrc
echo 'export HOMEBREW_BOTTLE_DOMAIN="https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles"' >> ~/.zshrc
echo 'export HOMEBREW_BREW_GIT_REMOTE="https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/brew.git"' >> ~/.zshrc
echo 'export HOMEBREW_CORE_GIT_REMOTE="https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/homebrew-core.git"' >> ~/.zshrc
source ~/.zshrc
```

### 1.7 TeXLive / LaTeX（写综述、编译 PDF 时用）

**Q：TeXLive 安装失败（"download did not succeed"）**
多为网络/镜像源问题。更换 CTAN 镜像（清华、中科大）后重跑 `install-tl-windows.bat --profile installation.profile`。更轻量的替代：**MiKTeX**（自动下载缺失包）。也可直接让 Claude Code 帮你诊断安装。

### 1.8 安装路径、C 盘空间与备份

**Q1：C 盘空间不足，软件必须装 C 盘吗？**（★）
不必。Miniconda、VS Code、Node.js 安装时都可自定义路径，建议装到 D 盘等非系统盘（如 `D:\miniconda3`）。**注意路径不能含中文**。50GB 一般够放开发环境（不含大项目和数据缓存），但需定期清理临时文件。

**Q2：用 AI 会不会把 C 盘塞爆？临时文件能清吗？**
AI 工具确会产生大量临时文件和缓存。可用 **WizTree** 可视化分析磁盘占用再清理；清理需谨慎，**别误删系统文件和 Claude Code 运行所需的配置**。

**Q3：给 Claude Code 充分权限后，怎么保护重要文件（如论文 PDF）？**
- Mac 推荐 **Time Machine + 外置大硬盘**做全量备份（外置传统硬盘性价比远高于苹果内置存储）。
- 重要文件定期备份；工作目录用 **Git 做版本控制**；不要把整个用户目录当工作目录。
- 坚果云备份 Zotero 基本可靠，但云端导出偶有文件打不开（概率较小）。

**Q4：Claude Code / Miniconda 装完提示"未找到命令""不是内部或外部命令"**
1. 检查环境变量：`where conda` / `where node` / `where npm`（Mac 用 `which`），为空则把对应目录加进 PATH。
2. **重启终端**（环境变量需重开终端生效）。
3. 仍不行则可能安装出错，卸载重装并注意勾选"添加至 PATH"。
4. 也可让 OpenCode 帮忙："帮我配置环境变量，让 conda/npm 命令可在终端中使用"。

**Q5：Google 账号被删除还能恢复吗？**
有官方恢复流程，按 Gmail 账号恢复链接的指引操作即可。

> **AI 辅助提示词（适用于本章所有安装报错）**：「我在 [Windows 11 / macOS] 上按连享会教程安装 [软件名]，运行 `[命令]` 时报错，完整报错如下：[粘贴报错文本/截图]。我已尝试 [列出已试方法]。请判断是网络、权限还是依赖问题，并给出可直接复制运行的修复命令，分步骤说明。」

## 第二章 · Claude Code 安装与配置

### 2.1 安装方法

**Q1：Claude Code 安装失败 / `irm https://claude.ai/install.ps1 | iex` 报错（"参数列表中缺少参量"、出现网页 HTML、"Just a moment..."）**（★★★★）
> 原因：PowerShell 没下载到真正的 install.ps1，而是把网页内容当脚本运行了。

**首选方案——用 npm 国内镜像安装（最稳）：**
```
npm install -g @anthropic-ai/claude-code --registry=https://registry.npmmirror.com
```
其他方案：①先装 Node.js（LTS）再 `npm install -g @anthropic-ai/claude-code`；②实在不行让助教提供安装包手动放入 `~/.claude`。装完用 `claude --version` 验证。

**Q2：通过镜像装的 Claude Code 版本是不是有点低？**（★）
镜像版本可能略旧但能正常用。要升级看 [2.3 版本与更新]。新手优先保证"能跑通"，版本不是首要问题。

**Q3：CC 下载困难，直接在 VS Code 装 Claude 插件可以吗？**
插件版功能基本一样，但**部分操作与课堂演示不同**，跟做会有障碍。建议仍用国内镜像装 CLI 版以保持一致。

**Q4：装好 OpenCode 了，还能再装 Claude Code 吗？会冲突吗？**（★）
能装，**两者不冲突**，各有独立配置和 skills 目录。两个都装的好处：CC 打不开/没配好时可用 OpenCode 来诊断修复。接国内模型时再装 cc-switch 统一切换。

**Q5：Claude Code 安装路径报错**
安装路径不能含中文字符，请放在非系统盘的纯英文路径下（如 D 盘）。

**Q6：怎么判断 Claude Code 装好了？**（★★★）
终端输入 `claude` 回车，**直到出现对话框**即为成功（没反应就再按一次回车）；或 `claude --version` 看版本号。装好后继续配 cc-switch 接模型。

**Q7：Windows 安装到第 8 步提示需要 sign in（登录）**（★）
这是正常步骤，教程要求做到第 9 步（登录授权）。**但如果你走"CC + cc-switch 接 DeepSeek"路线，则不需要在这里登录**——应通过 cc-switch 配置模型（见[第四章](#第四章--cc-switch-与模型-api-接入)）。

### 2.2 登录与模型接入（重点）

**Q1：有 Claude 会员后还要额外买 API 密钥吗？登录三个选项选哪个？**（★★★）
- Claude **会员订阅**（官网 20 刀/月起）主要用于网页版 Claude；**API 密钥**（在 console.anthropic.com 创建）是另一套独立计费。
- CC 登录时若用会员，要选 **"Subscription"**，不要选 "API Key"，否则会"登录成功却仍无法使用 / invalid api-key"。
- **如果你用 DeepSeek 等国内模型**：根本不用碰这三个登录选项，应通过 cc-switch 配置；配好后正常不会再弹登录界面，**弹了就说明 cc-switch 没生效**。

**Q2：装好 CC 后提示 "Not logged in. Please run /login"，没连上 DeepSeek**（★）
`/login` 是 Claude 官方账号入口，**不是接 DeepSeek 的地方**。先 <kbd>Ctrl</kbd>+<kbd>C</kbd> 退出，改用 cc-switch 图形界面统一接入模型（下载、教程见[第四章](#第四章--cc-switch-与模型-api-接入)）。

**Q3：CC 桌面版安装后要求认证，是不是装不了？**
不是。按"第三步"教程通过 cc-switch 接 DeepSeek 即可使用，**无需 Claude 官方账号认证**。

**Q4：VS Code 里输入 claude 跳转到网页 / 在哪运行更好？**
- 跳网页是**首次身份验证**，浏览器完成授权后，以后终端输入 `claude` 就不再跳转。用 cc-switch 接 DeepSeek 则不弹登录。
- 终端直接跑还是 VS Code 里跑，**功能完全相同**，看个人习惯。VS Code 好处是能同时看代码和终端。

**Q5：Claude Code 本身收费吗？它和 DeepSeek 是什么关系？是 "fake Claude" 吗？**
- CC 是客户端框架，**本身不收费**，收费的是接入的模型 API。
- 关系：**CC 是"手和脚"（跑命令、改文件），DeepSeek 是"大脑"（思考、回复）**。回复内容来自 DeepSeek，查文件/跑命令等动作由 CC 执行。这不是"假 Claude"，而是用 CC 这个优秀外壳驱动你选的模型。

**Q6：从 DeepSeek 换回原生 Claude，原来的 Skill 还能用吗？**
能。切换模型不影响已有 skills/plugins，只要它们仍在同一 `.claude` 目录或项目配置目录下。

**Q7：分析 PDF 时报 "Request too large (max 32MB)"**
内容超过 32MB 上限。用 `markitdown` skill 把 PDF 转成 MD 再发，或拆分大文件、只贴关键部分。

### 2.3 版本与更新

**Q1：新版 CC（如 2.1.156）对接 DeepSeek 报 `API Error 400 ... unknown variant 'system'`**
新版 CC 与 DeepSeek 的 message role 格式不兼容。**回退版本**：
```bash
npm uninstall -g @anthropic-ai/claude-code
npm cache clean --force
npm install -g @anthropic-ai/claude-code@2.1.121
claude --version
```
（MiniMax 同理需回退，可试 `@2.1.152`；DeepSeek 在 2.1.156 已兼容。）

**Q2：Mac 上 CC 一直提示已是最新版，怎么强制更新？**
Homebrew 默认不从 latest 通道更新：
```bash
brew uninstall --cask claude-code
brew install --cask claude-code@latest
```

**Q3：Desktop 版和终端版哪个推荐？**
推荐**终端（CLI）版**：更新最快、新功能最先体验、与课堂演示一致。

**Q4：装某一步报错怎么彻底重装？**
管理员身份打开 PowerShell 清残留后重装：
```powershell
Get-Process claude,node -ErrorAction SilentlyContinue | Stop-Process -Force
npm uninstall -g @anthropic-ai/claude-code
Remove-Item "$env:APPDATA\npm\node_modules\@anthropic-ai\claude-code" -Recurse -Force -ErrorAction SilentlyContinue
npm cache clean --force
npm install -g @anthropic-ai/claude-code --registry=https://registry.npmmirror.com
```

### 2.4 会话管理与常用命令

| 命令 / 功能 | 作用 |
| --- | --- |
| `/clear` | 完全清空上下文，从头开始 |
| `/compact` | 压缩上下文，**保留关键信息和大体计划**（压缩后 CC 仍记得） |
| `/context` | 查看上下文窗口已用比例（也可看终端底部状态栏，如 `17.4K(8%)`） |
| `/statusline` | 配置状态栏样式（建议含模型名和 context 使用率） |
| `/sessions` | 查看历史对话记录 |
| `/resume` | 恢复之前的会话（VS Code 重启对话不会永久丢失） |
| `/doctor` | 自诊断与修复（plugin/skill 问题、被丢弃的 skill 等） |
| `/btw` | 任务运行中插入**轻量补充询问**（不是 skill，也不能替代关键任务修改） |
| `/dangerous` | 赋予 AI 不询问直接执行的权限（遇不确定操作仍会暂停询问） |

**Q1：上下文很容易到限制，`/compact` 压掉的内容是不是就没了？**
不是。`/compact` 会总结保留关键信息和计划，压缩后仍可继续，需要时再用。

**Q2：运行中追加指令显示 "press up to edit queued messages"，一直排队不执行**
追加指令会进队列而非立即执行。建议**先中断当前任务**，把新增要求和原任务合并成一条更完整的指令重跑。`/btw` 只适合轻量旁路询问。

**Q3：长任务跑完怎么自动提醒我？**
在指令末尾加一句："完成任务后请用（文本/响铃）提醒我任务已完成。" CC 会照做，无需额外插件。

**Q4：CC 终端里公式怎么正确显示？**
终端不渲染公式。让 AI 把回答存成 Markdown 再用 VS Code 插件渲染；或在 Obsidian 装 Claudian 插件。

**Q5：CC 里的 think / think hard / ultrathink 等于网页版的 Deep Research 吗？**
不等于。它们属于"思考强度提示"，让模型本轮更认真推理，**不会触发联网检索+资料汇总+引用生成的 Deep Research**。CC 主要面向代码库读取、文件修改、命令执行等开发任务。

**Q6：CC 能和手机端联动吗？**
官方有手机 App 远程控制功能，但手机也需挂梯子，操作较麻烦。

### 2.5 Claude Code 综合

**Q1：CLI、Desktop、VS Code 插件，三者关系？**
底层模型相同、功能相近。**CLI 版最灵活、更新最快、与课堂一致**；插件版界面更直观但部分操作不同；Desktop 与插件版基本一致。框架差异远小于模型和指令带来的差异。

**Q2：用同一个模型，CC 和 OC 性能/效果一样吗？**
基本一样——**框架差异 ≪ 模型与指令差异**。但 CC 社区更热、生态更全、对 Claude 自家模型深度适配，遇到问题更易找到帮助。

**Q3：CC 装在日常电脑上安全吗？担心论文 PDF 被改坏**
只要不给过多权限就没问题：别把用户目录当工作目录、不授权直接操作系统文件、重要文件定期备份、工作目录用 Git 版本控制。AI 执行敏感操作前都会询问。

## 第三章 · OpenCode 安装与使用

### 3.1 安装与启动

**Q1：怎么打开/启动 OpenCode？**
在 PowerShell（或终端）进入工作目录后输入 `opencode` 启动它的文字界面（TUI）。

**Q2：`where opencode` / `Get-Command opencode` 无任何返回**
说明没加进 PATH 或没装成功。确认 `C:\Users\<用户名>\AppData\Roaming\npm` 下有 `opencode.cmd`，把该目录加进用户 PATH（此电脑→属性→高级系统设置→环境变量→用户变量 Path→新建→粘贴），关闭所有 PowerShell 重开测试。

**Q3：OpenCode 启动报错 / 路径或环境变量未刷新**
1. `where opencode` 确认实际调用路径，`opencode --version` 确认版本。
2. 关闭所有 PowerShell，打开一个**普通 PowerShell（不进 Anaconda base 环境）**再试。
3. 仍报错则清旧包重装：
   ```
   npm uninstall -g opencode-ai opencode-windows-x64 opencode-windows-x64-baseline
   npm install -g opencode-ai opencode-windows-x64
   ```

**Q4：`cd $env:USERPROFILE\opencode-lab` 提示找不到文件夹**
手动创建即可：`mkdir $env:USERPROFILE\opencode-lab` 再 `cd` 进去；也可 cd 到任意现有测试目录（如 `cd ~\Documents`）。

**Q5：OpenCode 到底是什么？是模型还是平台？**
是一个**可接入并切换多种模型的工作平台**，本身不是 AI 模型。

### 3.2 模型配置

**Q1：OpenCode 里没有 DeepSeek / Kimi，`/model` 找不到**
免费模型是**轮换变化**的，能正常交流就行。要稳定用 DeepSeek：用 cc-switch 配置（见[第四章](#第四章--cc-switch-与模型-api-接入)），或用 `/connect` 命令连接模型。

**Q2：`/connect` 选 DeepSeek Pro 后让选高/中/低，选哪个？**
选**"高"**，影响 effort level（思考深度），更高的代码质量和完成度。

**Q3：cc-switch 里配好了 DeepSeek，OpenCode 显示的还是 MiniMax / 其他模型**
配置生效后**需重启 OpenCode**。输入简单问候，看返回内容和底部模型名判断是否切换成功。

**Q4：页面和老师先导视频里不一样（免费模型不同）**
免费模型定期轮换，不用担心，能正常交流即可；配好 DeepSeek 官方 API 后会快很多。

### 3.3 使用与故障排除

**Q1：OpenCode 里无法复制粘贴、右键也不行**
- Windows：试 <kbd>Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>V</kbd>，或换 **Windows Terminal**（<https://apps.microsoft.com/detail/9n0dx20hk701>）。
- Mac：先**手打第一个字**，之后就能粘贴了。

**Q2：读取 PDF 报 "Invalid request Error"**
OpenCode 不能稳定直接读 PDF（尤其扫描版/图多/排版复杂/大文件）。**先把 PDF 转成 txt 或 markdown** 再读（用 markitdown）。

**Q3：连续问答报 "Unable to connect to huggingface.co"**
HuggingFace 网络问题。设镜像：
- Windows：`$env:HF_ENDPOINT = "https://hf-mirror.com"`
- Mac：`export HF_ENDPOINT="https://hf-mirror.com"`

**Q4：`/plugin` 显示 "no matching item"**
OpenCode **不支持 `/plugin`**。plugin-dev、marketplace 是 Claude Code 专属，OpenCode 没有 marketplace 概念。OpenCode 有自己的 MCP（`opencode mcp add`）和 skills 目录（`~/.config/opencode/skills/`）。

**Q5：<kbd>Ctrl</kbd>+<kbd>P</kbd> 呼出的是文件搜索，不是 Skill 菜单**
在 OpenCode 里输入 `/` 前加几个空格，可触发 skill 补全菜单。

**Q6：报"只能访问 C:\Users\Administrator 下的文件"（目录权限）**
用"A 修复 B"策略：先在 `C:\Users\Administrator` 下启动 OpenCode 确认能跑，再把报错描述发给 AI，让它诊断并修复另一目录的权限配置。

**Q7：在哪换 API Key / 嵌入 ChatGPT / 切换模型？**
- 换 Key：`/connect`。
- 切模型：`/models`（前提是已配好对应 API Key）。

**Q8：OpenCode 和 Claude Code 同时装会冲突吗？**
不会，两者独立、各有配置和 skills 目录，cc-switch 可统一管理它们的 API。

**Q9：想让简单任务用一个模型、复杂任务用另一个？**
用 CC 实现自动切换较复杂；OpenCode 的 **subagents** 机制可为不同子代理指定不同模型，更简便。

**Q10：在 VS Code 里打开 OpenCode 后怎么用？**
直接在终端用自然语言下达需求即可——让它解释当前文件、改代码、读项目结构、生成脚本等。

## 第四章 · cc-switch 与模型 API 接入

> **cc-switch 是国内学员接入模型的核心工具。** 下载：<https://github.com/farion1231/cc-switch/releases>；视频教程：<https://www.bilibili.com/video/BV1L7c7zZEAt/>。

### 4.1 cc-switch 概念与配置

**Q1：cc-switch 是什么？和 OpenCode / Claude Code 有何区别？**
cc-switch 是**配置管理 + 一键切换工具**，用来管理不同的 API Key / BaseURL / 模型名，方便在官方 Claude、DeepSeek、Kimi、GPT 等接口间快速切换。**它不是 AI 编程工具**——CC/OC 是干活的代理，cc-switch 负责管它们的 API 配置。

**Q2：装 cc-switch 就是为了给 CC 接 DeepSeek 吗？想用官方 Claude 模型怎么办？**（★）
是的，它能让 CC 方便地接国内模型。要用官方 Claude：在 cc-switch 里切回 **Anthropic 官方**，再在 CC 里 `/login` 登录。但国内用 Claude 官方 API 很快会被封，条件不允许时建议用 DeepSeek。

**Q3：cc-switch 里怎么配置 DeepSeek？（标准步骤）**
1. 打开 cc-switch，点右上角 **"+"**。
2. 在"预设供应商"里选 **"DeepSeek"**（**不要选 default**）。
3. 填入 API Key（DeepSeek 平台申请：<https://platform.deepseek.com/>）。
4. 模型映射：主模型 `deepseek-v4-pro`；Haiku 默认 `deepseek-v4-flash`；Sonnet 默认 `deepseek-v4-pro`；Opus 默认 `deepseek-v4-pro`。
5. 点 **"启用"**，到终端输入 `claude` 说"你好"测试。

**Q4：配置中转 API，API Key 填在哪里？**
**别用 "default" 预设。** 点 "+" 选具体供应商（DeepSeek、MiniMax 等）即出现 Key 填写框。自定义中转选"自定义"，填请求地址（API 格式选 **Anthropic Messages 原生**），在模型映射区填模型名。

**Q5：模型名里的 `[1m]` 要删吗？**
要。`deepseek-v4-pro[1m]` 里的 `[1m]` 是终端 ANSI 转义码渲染残留，**真实模型名是 `deepseek-v4-pro`**（不带 `[1m]`）。在 cc-switch 里直接写 `deepseek-v4-pro`。
（题外：有 Claude 会员时切回旧版中文模型可用 `/model claude-opus-4-6[1m]`，这里的 `[1m]` 是命令写法，与 cc-switch 填模型名是两回事。）

**Q6：cc-switch 里点"打开终端"一直让我选文件夹**（★）
**不需要**通过 cc-switch 的"打开终端"启动。配好并"启用"后，直接从开始菜单打开 PowerShell/终端输入 `claude` 即可。cc-switch 只是配置管理器。

**Q7：cc-switch 链接 DeepSeek 一直报错怎么修？**
关掉重开、重新生成 API Key 多试几次；**特别注意模型名大小写**（从官网复制可能全变小写导致不匹配）；老版本 cc-switch 卸载后下载最新版（链接见本章开头）。

**Q8：cc-switch 安装包在哪下？支持删除会话吗？**
访问 <https://github.com/farion1231/cc-switch/releases> 下拉找最新版（.msi 或对应系统格式）。删除会话功能在 v3.12.0 起支持，旧版请更新。

**Q9：cc-switch 安装的 Skills 在 VS Code 里调用方式一样吗？**
完全相同，与原生环境一致。

**Q10：想把 Skill 从 Claude Code 切到 Codex**
以管理员身份运行 cc-switch，把 Skill 文件夹（如 `cnki-paper-detail`）从 `C:\Users\用户名\.cc-switch\skills\` 手动复制到 `C:\Users\用户名\.codex\skills\`。

### 4.2 模型 API：概念、配额与费用

**Q1：AI 会员号和 API 是一回事吗？**
不是。**会员号**用于登录网页/App 人工聊天；**API** 是给程序调用的接口，需 API Key、按 token 计费。CC + DeepSeek 需要的是 **API Key，不是网页会员**。

**Q2：用 AI 必须有 API 吗？智谱要收费+实名，必须吗？**
不必非选智谱。可在 DeepSeek、智谱、Kimi、Anthropic、OpenAI 间自由选，cc-switch 可随意切换。**预算有限首推 DeepSeek**（性价比最高）。

**Q3：DeepSeek 最少充多少够课程用？**
**约 100 元**即可满足课程期间正常使用，余额很耐用。充值后在 cc-switch 填入 Key。

**Q4：怎么把 API Key 粘到 OpenCode？粘不进去**
- 复制：在网页 Key 旁点复制小图标。
- 粘贴：旧版 PowerShell 不支持 <kbd>Ctrl</kbd>+<kbd>V</kbd>，用 <kbd>Ctrl</kbd>+<kbd>Shift</kbd>+<kbd>V</kbd> 或右键粘贴；装了 Windows Terminal 后可直接 <kbd>Ctrl</kbd>+<kbd>V</kbd>。

**Q5：DeepSeek API 接入后一直 "Retrying..." / 响应慢 / 403 / "API key has expired"**（★）
逐项排查：
1. **确认是真实的 DeepSeek API Key**（不是示例 Key），且账户有余额、未过期（到 DeepSeek 平台确认）。
2. **关掉梯子/代理**——DeepSeek 是国内服务，开着梯子反而连不上；关掉后重启终端再试。
3. 用 **cc-switch 图形界面**核对 Key、BaseURL、模型名，避免环境变量配置出错。
4. 换网络节点或重启路由器。

**Q6：报 `deepseek-chat ... may not exist`**
DeepSeek 已停用 `deepseek-chat` 这个名字，改用 **`deepseek-v4-pro`** 并正确配 base_url。用 cc-switch 配置可避免拼写问题。

**Q7：智谱里没有 GLM-5 / 已付费却提示"余额不足或无可用资源包"**
- GLM-5 找不到：免费版模型列表阶段性更换，选可用的最新版本即可（参考课程配套页的推荐列表）。
- 已付费仍提示余额不足：检查三点——API Key 是否来自**同一个智谱账号**、base_url 是否填对、模型是否在当前套餐支持范围内。也可改用 DeepSeek（cc-switch 接入）。

**Q8：Kimi 额度没怎么用就报错**
Kimi 有 **5 小时滚动限额 + 7 天总限额**双重限制，即便额度没用完也可能因服务器负载过高报错。官方渠道速度和稳定性都一般，可换 DeepSeek/智谱。

**Q9：`/key` 设的模型和 `/model` 选的要一致吗？**
不需要。`/model` 选你想用的模型，`/key` 对应你 API Key 的服务商，二者独立配置。

**Q10：能让不同项目用不同模型配置吗？**
cc-switch 目前只支持**全局切换**。需要并行用不同模型，建议用 OpenCode 的 subagents 机制给各子代理指定模型。

**Q11：`/extra-usage` 是什么？要额外收费吗？**
超出套餐后转为按流量计费继续运行，**扣费很快**。不开则到额度上限 CC 会自动停下。

**Q12：工作台建好后调用信息还烧 token 吗？本地部署划算吗？**
工作台让**同等 token 费用下产出更高效**，并非免除 token 消耗。除非有数据保密硬需求，否则不建议本地部署——同等输出质量的本地部署需 10 万+硬件投入。

**Q13：中转 API 可靠吗？**
可去闲鱼货比三家，但**掺水问题目前难验证**。无 Claude 账号更建议直接用国内模型（cc-switch 接 DeepSeek）。

**Q14：image-gen Skill 的 API 怎么配？**
用 Google AI Studio：到 <https://aistudio.google.com/> 用谷歌账号登录、创建 API Key、绑卡即可使用。

> **AI 辅助提示词（API 报错通用）**：「我用 cc-switch 给 Claude Code 接 [DeepSeek/智谱/Kimi]，报错如下：[粘贴报错]。我的模型名填的是 [xxx]，已确认 [余额/网络/是否开梯子]。请帮我逐步定位：是 Key、BaseURL、模型名、版本兼容还是网络问题，并给出修正后的配置。」

## 第五章 · 网络与代理（科学上网）设置

> **核心结论**：接 **DeepSeek 等国内模型不需要梯子**；只有用 **Claude/ChatGPT 官方服务、访问 GitHub/HuggingFace** 等海外资源时才需要稳定代理，且通常要开**全局/TUN/虚拟网卡模式**。

**Q1：什么是"梯子"？怎么配科学上网？**（★★★）
"梯子"指 VPN/代理工具（Clash、V2Ray、Shadowsocks 等），用于访问海外服务。关键点：
1. **开启 TUN 模式（虚拟网卡）**，让系统全局流量走代理，而非仅浏览器代理——CC/OC 需要全局代理才能连。
2. 选**稳定节点**、固定地区（美/日），别频繁切换。
3. 先验证代理：能否打开 <https://www.google.com> / api.anthropic.com / claude.ai。

**Q2：挂了梯子还是连不上 Claude / 报 `Failed to connect to api.anthropic.com` / `ERR_BAD_REQUEST`**（★★★）
按三层排查：
1. **地区**是否支持 Anthropic 服务。
2. **代理是否真生效**：开 TUN/虚拟网卡；`Test-NetConnection api.anthropic.com -Port 443`（Win）或 `curl -v https://api.anthropic.com` 测连通；`ERR_BAD_REQUEST` 常是**节点不干净**，换干净节点。
3. **冲突**：是否多个代理客户端同时运行、是否被防火墙/校园网/杀软拦截。
> **最省事的替代**：通过 cc-switch 接 `deepseek-v4-pro` 绕过所有 Anthropic 网络问题。

**Q3：开了全局 TUN，关掉梯子就上不了网了**（★）
TUN 接管了系统网络，关代理前应在客户端里先关闭 TUN/系统代理，或退出客户端时让其自动恢复系统设置。若已断网，重启代理客户端→关闭 TUN→退出，网络即恢复。

**Q4：加载了代理环境变量后 OpenCode/DeepSeek 反而连不上，删掉就好了**
`HTTP_PROXY`/`HTTPS_PROXY`/`ALL_PROXY` 会**强制 DeepSeek 也走代理**导致异常。若不用国外模型、且删掉这些变量后仍能正常访问外网，就删掉它们；装了 VPN 直接打开客户端即可。推荐用 cc-switch 管理，免去手动设环境变量。

**Q5：换了好几个代理还是这个问题 / 截图网络报错**
先确认开了 **TUN/虚拟网卡**且只有一个代理客户端在跑；仍不行就**改走 cc-switch + 国内模型**绕开。

**Q6：之前一直稳定，今天突然连不上 API**
多为网络波动或代理冲突。重启网络、用 cc-switch 重配 API、确保只有一个代理在运行。

**Q7：访问 GitHub 失败（加 marketplace、clone 仓库、下 Skills）**
GitHub 在国内需科学上网才稳定。替代：手动下载仓库 ZIP / 公众号离线安装包，再让 AI 按本地路径离线安装。

**Q8：ChatGPT 需要科学上网吗？**
需要。

**Q9：Claude 会被封号吗？怎么降低风险？**
近期 Anthropic 管控趋严，**频繁切节点、不干净代理、异常支付/IP** 都会增加封禁风险。降低风险：用稳定高质量梯子、别频繁切换、付款方式和 IP 正常。
- 用 **cc-switch 接国内模型（DeepSeek 等）无封号风险**。
- 在 VS Code 里用 CC 本身不会导致封号；分析**中文文献**只要付款方式和 IP 正常也不会被封。

> **AI 辅助提示词（网络通用）**：「我在 [系统] 上用 [代理工具]，已开/未开 TUN 模式。访问 [google.com / api.anthropic.com / github.com] 的结果是 [能/不能]。Claude Code 报错：[粘贴]。请判断是地区限制、代理未生效、节点不干净还是端口/防火墙问题，并给出最简解决方案；若复杂，请直接告诉我怎么改用 cc-switch + DeepSeek 绕过。」

## 第六章 · Skills 与 Plugins 管理

### 6.1 Skills 安装与使用

**Q1：Skills 在哪下载/怎么安装？**（★★★）
- 来源：解压"粉丝福利 Skills.zip"（含约 20 个 skill）、"李学恒老师.zip"里的 skills 文件夹，或指定 GitHub 仓库。
- **最省事**：解压后在 Claude Code 里直接说"帮我安装这些 skills"（给它本地路径或网址即可，Agent 会自动处理安装位置）。
- 手动：把 skill 文件夹复制到 `~/.claude/skills/`（Windows：`C:\Users\<用户名>\.claude\skills\`）。
- 安装时遇到问题，可直接把 `recommended_skills` 下的文件夹剪切到 `skills` 文件夹；从 GitHub 装则需开启科学上网（TUN/虚拟网卡）。

**Q2：通用安装路径怎么写？`~/.claude/` 对吗？还是放在哪？**（★）
**一般不用手动指定路径**——把网址或本地文件夹丢给 Agent，让它"安装这些 Skill"，它会自动放到位。手动放则用 `~/.claude/skills/`。

**Q3：装完 `/skills` 显示 "No skills found" / 没显示列表**（★）
Skills 需会话**重启后**才被加载。关掉 Claude Code 重新进入，再输入 `/skills`。

**Q4：CC 和 OC 的 Skills 能共用吗？同一个 Skill 两边跑会互相干扰吗？**
- CC 默认读 `~/.claude/skills/`，OC 默认读 `~/.config/opencode/skills/`。把同一批 skills 复制到 OC 对应目录即可（**OpenCode 官方也支持读取 `~/.claude/skills/<name>/SKILL.md`**，多数标准 skill 可直接用）。含 CC 专属命令（如 `/plugin`）的需改写适配。
- 大多 Skill 本质是提示词+流程说明书，**多个窗口/工具同时读同一 Skill 各干各的，不会互相干扰**。
- 同时装 CC 和 OC，**Skill 装一份到对应目录即可，不必重复"安装"**。

**Q5：Skills 装在全局有风险吗？**
建议先在**单个项目目录**测试，确认来源可靠、代码清楚、不会乱读写文件，再放全局。来自可信来源（GitHub 有明确开源授权）的 skills 风险较低。

**Q6：怎么把 Skills 装到 U 盘随身用（项目级）？**
Skills 支持项目级目录。把 skills 文件夹放 U 盘，用 `--plugin-dir` 指定：`claude --plugin-dir "U盘的skills路径"`。Skill 的"级别"取决于放置位置——项目文件夹下=项目级，电脑根目录文件夹下=全局级。

**Q7：怎么编辑/优化已有 Skill？SKILL.md 和 reference 目录是什么关系？**
- 编辑：直接在 VS Code 改对应 `SKILL.md`；更简单是把 SKILL.md 发给 AI，说想怎么优化，让它生成新版本。
- 关系：**根目录 `SKILL.md` = 指挥中心**（定义工作流程、决定 AI 怎么想怎么动）；**`reference/` 里的文件 = 工具箱**（提供素材、模板、标准）。改大流程动 SKILL.md，改细节动 reference 模板。

**Q8：怎么把 PDF 批量转成 MD？**
用 **markitdown** skill，在 CC 里输入：`/markitdown 帮我把当前文件夹下所有 PDF 都转成 MD 格式`。不用预先下载，直接说"帮我安装 markitdown 这个 skill"即可。若调用报不存在/自动 pip install：把 markitdown 与待处理文件放同一目录，再 `@` 引用 markitdown 明确说"调用这个 skill 把 xxx 转成 yyy 格式"。

**Q9：常见 Skill 在哪找？**
- **Five Questions（五问拆解）/ academic-paper-search**：在"李学恒老师.zip"或"粉丝福利 Skills.zip"的 skills 文件夹内。
- **知网 CNKI**：GitHub <https://github.com/cookjohn/cnki-skills>（把 URL 给 CC 让它安装，或群文件下 `cnki-skills-master.zip`）。

**Q10：终端加载时丢掉很多 skills（VS Code 里却正常）**
提示 skill descriptions dropped 说明 description 总量超出会话预算。运行 `/doctor` 看受影响的 skills，精简 description 或把低频 skill 设为 name-only。

**Q11：Kimi 调用 skill 时不好好干活**
Kimi 执行 skills 的自觉性较差。反复强调"必须使用指定 skill"，或让 AI 自行写脚本实现该 skill 功能。

**Q12：`/skills` 列表左边的"锁"图标是什么？**
代表**权限锁定**：该技能涉及高风险操作（删文件/联网），AI 调用前必须经用户手动授权；该 skill 为系统内置/受保护，源代码不能在终端内直接改。这是 CC 的安全机制。

**Q13：`/btw` 是 Skill 吗？**
不是。它是 CC 内置交互功能，用于在任务运行中插入轻量补充询问。

**Q14：有没有语音转文字+生成总结的 Skill？**
有成熟方案：在 CC 对话框输入 `/voice` 开启原生听写，或接入第三方 MCP 用 Whisper 转录存本地；转好后一句话让大模型对文本做提炼总结。适合访谈、课堂笔记、头脑风暴。

**Q15：Wiki 知识库原始文件必须存本地吗？**
不必，也不必全加载进内存。更合理：原始文件放云服务器/对象存储/网盘，本地或数据库只存文档 ID、路径、元数据、文本切片和向量索引。但云存储只解决"存放"，真正决定效果的是**解析、切片策略、向量检索和权限管理**。

### 6.2 Plugins 安装与开发（Claude Code 专属）

**Q1：怎么理解 Plugin？它最终怎么用？**
Plugin = **配置好的 agents 文件 + skills 文件**，是一套自动化工作流。装好后，你在对话里描述任务，Plugin 定义的 subagent 会自动激活、拆解执行。本质就是一堆文件（SKILL.md、agent 定义、hook 配置），**把文件夹复制到目标平台对应目录就能用**。

**Q2：`/plugin marketplace add ...` 没反应 / 无法连 GitHub（"Failed to clone... Connection was reset"）**
这一步要连 GitHub，**必须先科学上网**（确认能访问 GitHub）。或浏览器下载仓库 ZIP，解压后用 `claude --plugin-dir "解压路径"` 加载。

**Q3：安装 plugin-dev 的正确流程？找不到 Marketplaces 标签**
1. 切到 **Marketplaces** 标签，添加市场 URL：`https://github.com/anthropics/claude-plugins-official`。
2. 回 **Discover** 标签搜 `plugin-dev` 安装。
3. 装完若提示 "Run /reload-plugins to apply"，按 ESC 退出搜索框，运行 `/reload-plugins`。
4. 连不上 GitHub：开代理/换节点，或下载 ZIP 后 `claude --plugin-dir "C:\解压路径\plugin-dev"`。

**Q4：OpenCode 没有 plugin-dev / 怎么对应 Plugin 功能？**
OpenCode 没有 Plugin 系统，用 **Skills** 实现类似功能：把 Claude Code plugin 文件夹里的 `skills/` 部分复制到 OpenCode 的 skills 目录；含 CC 专属命令的需改写。也可把整个 plugin 文件包发给 OpenCode 让它读取改写适配。

**Q5：怎么自己开发/修改 Plugin（如加"不要用破折号"规则）？**
找到并编辑对应 plugin 的 SKILL.md，或直接把修改需求发给 CC 让它改。更系统的方法：装好 plugin-dev 后用 `/plugin-dev:create-plugin` 创建新 plugin。

**Q6：Plugin 迁到 OpenCode/Codex 需要先在 CC 里装吗？**
不需要。Plugin 就是一堆文件，直接把整个文件夹复制到目标平台对应目录即可（不需科学上网）。注意 plugin-create 功能专门针对 CC，迁到 OC 需改写适配。

**Q7：安装 copy-edit-master 等 Plugin 报 "Path does not exist" / 安装失败**
用**实际的绝对路径**，避免空格和中文文件名；把 plugin 放纯英文无空格路径，或直接把 ZIP 发给 CC 让它自动装。失败时用 `/doctor` 让 CC 自诊断修复。

**Q8：Plugin / Agent / Skill / Subagent 是什么关系？**
- **Plugin** = agents 配置文件 + skills 文件的打包。
- **直接跟你对话、回复你的是"主代理"**；运行中看到 "Agent(派去干某活)" 即主代理派出的**子代理(subagent)**。
- "agent" 有三层含义：①广义=AI 工具与工作流集合；②静态的插件 agents 配置文件；③会话中独立的智能体单元（主/子代理）。

> **AI 辅助提示词（Skills/Plugins 通用）**：「我想在 [Claude Code / OpenCode] 里安装并使用 [skill/plugin 名]，它在 [本地路径/GitHub 网址]。请帮我完成安装并验证：装到哪个目录、需不需要重启、怎么确认 `/skills` 里能看到它；如果连不上 GitHub，请改用离线（ZIP / --plugin-dir）方式，给出可直接运行的命令。」

## 第七章 · MCP 服务器配置

> **MCP** = 让 AI 连接外部工具/数据库/接口的标准。常用：filesystem、stata、zotero、tavily、Chrome DevTools 等。

**Q1：Skills 和 MCP 有什么区别？装了 Skills 还要 MCP 吗？**
两者**不是替代关系**：
- **Skills** = 教 AI"怎么做事"的方法包（流程、Prompt 模板），适合联网查资料类工作流。
- **MCP** = 让 AI 稳定连接外部工具/数据库/API 的接口。
- **最佳搭配**：MCP 提供工具连接，Skills 教 AI 正确使用这些工具。

**Q2：OpenCode 提示"未配置 MCP 服务器"（No MCP servers configured）**
让 AI 帮你配（在 OpenCode 里描述需求），或用交互命令 `opencode mcp add`（在终端运行）。

**Q3：推荐哪些 MCP？要付费吗？**
- **Tavily MCP**：每月 1000 次免费搜索，**无需付费、不需梯子**，推荐装。
- **Zotero MCP**：连文献库。
- **Stata MCP**：按需装，用到再处理也来得及；提前配好上课直接用。
- **Chrome DevTools MCP**：需要自动下载受权限限制的中文文献（如知网）时用，控制浏览器。注意：开启远程调试会让外部应用完全控制浏览器，**仅在信任的应用中使用**。

**Q4：Zotero MCP 的 Library ID 就是 User ID 吗？怎么找？**
是的。
- 在 **Zotero → 设置 → 同步 → 数据同步** 查看 User ID。
- 或用 API 查：访问 `https://api.zotero.org/keys/YOUR_API_KEY`（替换为你的 Key），返回 JSON 里的 `userID` 字段。
- 本地 API（端口 23119）通常用 `/users/0/items`。
- 配置时告诉 AI："帮我配置 Zotero MCP 写入权限，API Key 是 [你的 key]，Library ID 是 [你的 ID]"。

**Q5：网页上找不到 Zotero 的 Key / 点不开 Settings**
API Key 在 **Settings → Security** 创建。点不开 Settings 时，先到 **Email** 页面完成邮箱验证（填验证码）。

**Q6：Zotero 连大模型读的是摘要还是全文？**
默认只读题名、作者、年份、摘要、标签、笔记等元数据。若插件支持读 PDF 附件、且 PDF 已下载到本地并有文字层，可读正文（通常分段检索）。

**Q7：Zotero 本地 API 连 localhost 报错**
Windows PowerShell 里 `curl` 实际是 `Invoke-WebRequest`，行为与标准 curl 不同。改用 `curl.exe` 测试：
```bash
curl.exe "http://localhost:23119/api/users/0/items?limit=1"
```

> **AI 辅助提示词（MCP 通用）**：「我想在 [CC/OC] 里配置 [Zotero / Stata / Tavily / Chrome DevTools] MCP，用途是 [说明]。我已有 [API Key / 本地端口 / Library ID]：[填写]。请给出完整配置步骤和验证方法；报错就贴给你：[粘贴报错]。」

## 第八章 · 模型选择与对比

> 模型与版本号信息来自课程答疑（2026-06），随时间会变；选型时以"自己已有、额度正常、连接稳定"为先，先把流程跑通，再追求最优。

### 8.1 国内 vs 国外、怎么选

**Q1：国产模型买哪个？DeepSeek / Kimi / GLM / MiniMax 怎么选？**
**首推 DeepSeek**（便宜好用，约 100 元很耐用）。其他：Kimi 有 coding plan（199 元/月一档较稳）、智谱 GLM 有 coding plan（<https://www.bigmodel.cn/glm-coding>）、MiniMax 也提供 API。**优先选你已有 Key、额度正常、连接稳定的那个**，先把流程跑通。

**Q2：有 GPT 会员，直接用 Codex 行吗？哪个最稳妥？**
- OpenAI 20 刀会员的 Codex **每 5 小时额度不够跑完一次完整文献综述**，会中途限额中断。
- 实测同一指令让多模型生成文献综述：GPT-5.5 中断两次才完成，且综述长度与文献覆盖比其他模型差一档。
- **预算充裕** → Claude Opus 4.7（公认最强但成本极高，Max 5x 套餐下 5 篇文献约耗 30% 五小时限额）；**预算有限** → `deepseek-v4-pro`（实测跑 60 篇文献总结效果很好，花费仅约 2 元）。

**Q3：上课能用 Codex 替代 Claude Code 吗？没有 CC 账号会跟不上吗？**（★）
可以用 Codex，**核心功能与交互逻辑基本一致**，但部分配置文件位置和插件系统有差异、需要做转换。**老师演示主要用 CC，跟做效果更好**。注意：以国内模型为主时不建议 Codex，因很多厂家未适配 Response 接口。

**Q4：国内外具体推荐型号？**
国内：GLM coding plan、DeepSeek。国外：GPT、Claude（如 gpt-5.4-mini、Gemini 3.1 Flash）。课程演示用 GPT-5.5 的是 20 刀/月 Plus 会员。

### 8.2 中文写作、正文输出怎么选

**Q5：中文写作哪个最好？Kimi / DeepSeek / Qwen / Claude**
中文自然度大致：**Kimi > DeepSeek > Claude Opus 4.6**；Qwen3 Max 中文表现令人惊喜。
- 主力推荐 **Claude Opus 做逻辑推理与框架搭建**，**中文写作场景用 DeepSeek 打磨**或回退到 Opus 4.6：`/model claude-opus-4-6[1m]`（**4.8 中文表现明显退化**，有学员反映 4.6 写的综述措辞高一档）。
- `fix-chinese` 润色优于原始输出，适合公众号级别，但离论文标准尚有差距。

**Q6：Opus 和 Sonnet 在正文输出里怎么选？**
**Opus** 适合逻辑推理、框架搭建；**Sonnet** 适合书写输出。中文写作仍推荐 DeepSeek 或回退 Opus 4.6。

### 8.3 额度、成本与稳定性

**Q7：太费 token 了，上午就花掉十几块**
- 考虑国产模型的 **coding plan**（按月比按量经济），如 Kimi 199 元档。
- **简单操作用基础模型**（如 `deepseek-v4-flash`），复杂任务才上 pro 模型。
- DeepSeek 当前便宜，先充 100 元试用。

**Q8：Claude 20 刀套餐够用吗？**
可切 **Sonnet** 模型省额度；DeepSeek 先充 100 元即可。配好后先交互几轮测试，确认无误再投入实际任务。

**Q9：Kimi 不稳定 / 换 GLM Max 好多了？**
Kimi 近期算力不足、服务器频繁过载；**智谱 GLM Max 表现稳定**，可作有效替代。国内模型选择需按各家当前服务状态灵活调整。

### 8.4 能力边界与判断

**Q10：怎么判断模型优劣？**
没有捷径，靠**归纳法**——通过大量实际交互积累经验和直觉。光看厂商宣传和跑分看不出来。

**Q11：终端输出质量不如网页？AI 还老瞎说（幻觉）？**
- 终端对 PDF/图片识别能力弱于网页版/客户端（网页版 OCR 更强）。
- AI 偷懒和幻觉是普遍现象，需**反复强调、加强语气、给出明确标准**来约束。

**Q12：课件里的"问问 AI"用什么模型？**
接入的是 **DeepSeek**。

**Q13：本地大模型部署实用吗（如 5090 + Qwen 3.6 27B）？**
本地模型能力有限、场景窄，**干不了 Agent 的活**。一张 4090/5090 不如一台 128GB 统一内存的 Mac Studio 实用。想试可从 Ollama 起步，直接在 CC 里让它调研方案和教程即可完成部署。

**Q14：教程配图用什么模型生成？中文乱码/不清晰怎么办？**
- 优先 **GPT Image 2**（`gpt-image-2`，在文字渲染、多语言、清晰度上明显更好；部分组织需先完成 API Organization Verification 才能调用）。
- **Nano Banana** 中文文字是短板（易错别字、笔画错乱），它更擅长构图质感；若用它，提示词强调"简体中文、无错别字、大字号、高对比度、锐利清晰"，避免太小太密的文字。
- 兜底：后期手动修正中文文字。

> **AI 辅助提示词（选型）**：「我的预算约 [金额]，主要任务是 [文献综述/中文论文写作/代码/数据分析]，已有 [DeepSeek/智谱/Claude/GPT] 账号。请基于性价比、中文质量、稳定性给我一个推荐模型与档位，并说明什么情况下该切换到更强/更便宜的模型。」

## 第九章 · Agent 工作流与高级用法

**Q1：配好模型后怎么"交互几轮"测试？（入门）**
交互=向 AI 发指令、看回复、再追问。先输入一个简单任务（如"帮我总结这篇文献摘要"），看输出再追问或修正。遇到问题直接把问题描述发给 AI 让它协助。

**Q2：`/do-agent` 怎么用？为什么老师特别看重它？**
`/do-agent` 让 AI 进入 **Agent 工作流模式**：先 Plan 规划、拆分步骤，再分别执行、自查纠错、汇总——像人一样在目标驱动下闭环做事，复杂任务更稳定，不用你一步步拆指令。基本用法：
```
/do-agent 写一篇系统专业的关于[主题]的文献综述。要求语言：简体中文。最后要用生成完整的 tex 和 bibtex 文件并测试生成 pdf 直至修复所有错误。
```
> 价值在于：把普通"被动单点问答"升级为"主动规划+分步执行+自我校验"的专业任务模式。复杂任务时在提示词前加 `/do-agent` 即可。

**Q3：调了五问拆解 skill，还要再调 do-agent 吗？**
看任务范围：**只拆解单篇文献** → 直接用 `five-questions` skill；**完整流程**（多篇文献+写综述+编译）→ 用 `do-agent`，它会自动规划并调用五问作为子步骤。

**Q4：LLM 工作流拆解的逻辑？sub-Agent 怎么各用不同模型？**
核心：把复杂任务拆成多个子任务，让不同 sub-agent 分别处理再整合。例如文献综述全流程：搜索文献 → 下载 PDF → 转 MD → 逐篇五问拆解 → 整合分析 → 撰写综述 → 编译 PDF → 修复错误。每步可用不同模型以优化成本质量。给子代理指定不同模型：用 **cc-switch** 统一管理（CC 全局切换），需并行不同模型时用 **OpenCode 的 subagents**。

**Q5：subagent 设多少个合适？有上限吗？**
没有硬上限，但太多难维护：**定义控制在 10–15 个以内，同时运行 3–5 个以内**；超过 8 个并行通常不再提效，反而烧额度、增混乱。并行 subagent 适合相互独立的调查任务；很多 subagent 同时返回详细结果会显著消耗主对话上下文。

**Q6：do-agent 跑完想继续完善怎么接？**
直接反馈让它继续改。若上下文已很长，先 `/compact` 压缩保留关键信息和计划，再继续。

**Q7：do-agent 改 research proposal / 短篇内容效果不好？**
do-agent 非万能，关键是**给明确的标准和方向**。技巧：先让 AI 做网络搜索调研、从权威来源总结出"什么是好标准"，再按标准制定计划执行。**AI 是放大器，要给它标准和边界**。

**Q8：AI 跑批量长任务半夜卡住不干了怎么办？**
把任务、计划、结束标准都写入文档，让 AI **边做边逐项打勾**，明确"不勾完不要停"；开始时反复强调"全自动干到底"，设清晰目标和结束条件。

**Q9：怎么让 AI 跨对话记住错误、建"错题本"（长期记忆）？**
建一个外部知识库当 AI 的"长期记忆"：在工作目录建 `error_log.md` 或 `instruction.md`，每次 AI 出错就让它把"错误案例+正确解法+避坑指南"追加进去；新对话开始先下令"请先阅读并严格遵守工作目录下错误日志中的所有规范"。也可把这些经验写进项目自定义系统提示词，成为该项目的原生准则。

**Q10：怎么学别人公开发布的工作流？**
关注优秀研究员/团队的 GitHub 仓库，Fork 到自己账号，让 AI 解释仓库的功能和用法。**核心是理解设计逻辑，而非简单复制。**

**Q11：创建审稿 Agent 后没出现对应的 md 文档？**
确认操作：按方向键下选 **"confirm and save"** 再回车。若 webfetch 的网址打不开，先在浏览器确认该网址能正常访问。

**Q12：Wiki 知识库真能"知识复利"、克服 RAG 局限吗？**
价值在于把零散材料/经验/卡片/常见问题沉淀成**可检索可复用的结构化资产**，长期有复利效应，尤其适合课程、研究项目、团队协作和反复迭代写作。但别神化：Wiki 只是改善 RAG 的上游工程，能缓解资料混乱、切片差、上下文缺失、重复检索，却不能彻底克服 RAG 局限。真正决定效果的是资料清洗、目录结构、元数据、切片策略、更新机制和人工校验。

> **AI 辅助提示词（工作流）**：「我要用 Claude Code 完成一个复杂科研任务：[描述任务，如'对某主题做文献综述并编译成 PDF']。请用 `/do-agent` 思路帮我：①拆解步骤；②为每步建议合适模型/skill；③写明每步的'好标准'和结束条件；④把易错点写进一个 `instruction.md` 让你后续遵守。」

## 第十章 · 文献检索、学术写作与论文发表

### 10.1 文献综述

**Q1：用 AI 写一篇专业文献综述的指令模板？**
基本命令：
```
/do-agent 写一篇系统专业的关于[主题]的文献综述。要求语言：简体中文。最后要用生成完整的 tex 和 bibtex 文件并测试生成 pdf 直至修复所有错误。
```
更完整的提示词应包含：「梳理文献研究脉络、方法分歧与核心发现，**按主题板块归类、不要逐篇罗列**；用 LaTeX 撰写，配套完整 `references.bib`（natbib+apalike），正文规范用 `\citet/\citep`；写好后调用 latexmk/xelatex 编译生成 PDF，遇错自行排查修复直到成功。」

**Q2：文献综述要不要写很详细的 Prompt？**
用课程设计好的 workflow/skill 时，Skill 内部已含写作要求（如"按主题归类、不逐篇罗列"）。**自己单独写 prompt 时，务必写清**写作风格、结构、引用规范等。

**Q3：一次性生成 vs 分步生成，哪个好？**
**复杂任务分步通常更优**。推荐流程：搜索文献 → 下载 → 转 MD → 逐篇拆解 → 整合 → 撰写。用 do-agent 自动拆解执行。长文本任务上，`deepseek-v4-pro`、Claude Sonnet 4.6 表现较好。

**Q4：给了 PDF 原文 AI 仍断章取义，怎么优化引用准确性？**
1. **先把 PDF 转 Markdown**（markitdown）——MD 让 AI 更准确读取文本结构。
2. **五步法**：先用 `five-questions` skill 逐篇拆解，再写综述。
3. 文献多就**分批处理**，每批控制数量确保 AI 仔细读每篇。
4. 推荐流程：PDF→MD → 五问拆解 → 生成 `search-results.md` → 基于 MD 写综述。

### 10.2 PDF 处理与文献下载

**Q5：编译 PDF 失败："本地无 LaTeX 环境 / 没装 BibTeX"**
LaTeX 环境（TeXLive）没装好。安装 TeXLive（下载失败就换 CTAN 镜像：清华、中科大），或让 AI 帮你诊断安装。

**Q6：AI 能读扫描版 PDF 吗？怎么 OCR？**
扫描版需先 OCR。方案一：让 CC 自动装 Tesseract+Poppler，用 Python 脚本 OCR 后转 Markdown。方案二：用 WPS 会员 OCR（淘宝约 1 元/天），几百页的书先用 WPS 转好再交给 AI。

**Q7：转 MD/Word 后 PDF 里的图片丢了，怎么保留图片？**
装 **Mineru AI Skill**（<https://clawhub.ai/mineru-extract/mineru-ai>），用前到 <https://mineru.net/apiManage/token> 申请 token。只有一两份文档可直接在 Mineru 网站手动转换以保留图片。

**Q8：下载到的多是 MDPI，几乎没有 Elsevier/IEEE 的 PDF？**
MDPI 是开放获取（OA）所以随便下；Elsevier/IEEE 全文在付费墙后。解决：用**学校邮箱**去 <https://dev.elsevier.com> 和 <https://developer.ieee.org> 各注册一个 API Key（前提是学校已订阅相应数据库，**校园网下注册会自动绑定机构权限**），把 Key 给 CC，它会写脚本调用两家 API 搜索和下载全文。**无机构订阅的学校不适用**。

**Q9：搜索/下载文献的提示词或工具推荐？**
- <https://github.com/openags/paper-search-mcp>：支持 arXiv、PubMed、bioRxiv 等多源搜索和 PDF 下载。
- <https://github.com/Imbad0202/academic-research-skills>：从研究到发表的全流程 skill 套件，内置引用验证和幻觉检测。
- 中文文献下载用知网 skills（cnki-skills）。

### 10.3 论文写作、润色与"去 AI 味"

**Q10：用 AI 写的中文质量差、AI 味重，怎么办？**
有效策略：①让 AI 从《经济研究》《中国社会科学》等权威期刊读几篇论文**总结语言标准**；②**多轮反馈迭代**，描述具体标准而非笼统要求；③用 `/do-agent` 自动迭代；④**必须给明确的格式、语言、风格标准**——AI 天然不擅长这些，不告诉它标准它写不好。

**Q11：怎么去 AI 味？有专门工具吗？**
`copy-edit-master` 等润色工具能改善文风，但**不能完全去除 AI 痕迹**。推荐流程：AI 生成初稿 → 润色工具改 → **人工审核调整**。可试开源工具：<https://github.com/blader/humanizer>、<https://github.com/shyuan/writing-humanizer>（让 CC 帮你下载安装）。降 AI 工具能降检测分，但改出来常读着别扭，仍需自己多思考编辑。

**Q12：用 Skill 润色后能直接投稿吗？**
不能一劳永逸。去 AI 味是**程度问题而非绝对**，学术界也没有统一自动判断标准。Skill 是提效辅助而非质量保证：AI 负责从 0 到 1 的初稿和结构化修饰，**作者仍需在最后环节注入人类学者的思考**，对核心因果逻辑和学术贡献做人工微调后再判断是否达到投稿状态。

**Q13：有没有针对中文论文修改的 Plugin？**
`copy-edit-master` 可用于中文学术润色；中文文献下载分析用 `cnki-skills`。要更特定的指导，可自定义 skill，把中文写作规范（如不用破折号、避免某些句式）写进 SKILL.md。

**Q14：批量批改学生论文有什么经验？**
用 `copy-edit-master` 类润色 plugin 批处理；把评分标准和修改要求**编写成 skill**，对每篇设统一检查标准，让 AI 逐篇输出诊断报告，最后结合人工审核。

**Q15：参考文献怎么适配不同期刊格式？**
不必先做"全能列表"。**提供 1–2 条目标期刊的参考文献格式样例**，AI 即可自动学习并适配整份列表。

**Q16：有没有做元分析（Meta-analysis）的 Skill/Plugin？**
群内暂无专门的元分析 skill。可找功能类似的现成 plugin 适配（比从零开发高效），或定制一个专用 skill。

**Q17：AI 提取准确率有公认阈值吗？**
没有统一阈值，取决于任务、数据和模型。可结合人工校验样本算精确率/召回率；学术文献通常需人工抽样验证关键信息。

**Q18：APE（自动化提示词工程）说得越多越好吗？**
不是。APE 核心是**用算法自动优化提示词**而非人工堆砌冗长指令。**提示词并非越长越好，关键在精准。**

> **AI 辅助提示词（写作通用）**：「请扮演[经济学/管理学]领域资深审稿人。先从《经济研究》等权威期刊总结 3 条中文学术写作的语言标准，再据此润色我下面这段文字，去除 AI 味、保持严谨学术语气，并列出你修改的依据。原文：[粘贴]。」

### 11.1 资料获取

3. 也可直接私信获取，助教会单独发。

**Q2：找不到"步骤 A-B-C"文档 / 新编书 / GitHub 资料**（★★★）

- 部分文档是老师新编书的一部分，内容与群内资料几乎一致，可先对照现有资料。
- 部分资料需在指定 GitHub 仓库自行下载，例如：<https://github.com/prashgarg/CausalClaimsInEconomics>。

**Q3：先导课视频对应哪个文档？**（★）
配套推文为《从 OpenCode 到 VS Code 到 Skills：一次配齐整套 AI 生产力工具》。

**Q4：怎么打开课堂提到的网址打不开？**
直接**点群里助教发的链接**而非手动输入；手动输入时要带完整协议名 `https://`。示例链接：`agent-book-beta.xueheng.site/chapters/chapter-3`。打不开就检查网络或换浏览器。

### 11.2 回放与设备

**Q5：课程有回放吗？观看条件？**（★★）
- **测试课无回放，正式课有回放**；回放约在每天直播结束后 **1.5 小时**生成，通过课程平台查看。
- **观看回放必须用与直播登录时相同的设备**，否则无法播放。

**Q6：能换设备（手机↔电脑）看吗？**（★★）
课程平台通常**绑定一台设备，开始上课后不能更换**，请在首次观看前确认设备合适。测试期间若要换设备，先退出当前设备再在新设备登录，并务必在正式上课前完成测试。特殊情况私信助教咨询能否解绑。

**Q7：测试时间是什么时候？**（★）
一般安排在开课前一天至开课前，**以群公告为准**。本次课程测试时间为 **6 月 18 日 ~ 19 日 9:00 ~ 23:00**。错过可联系助教补测。

### 11.3 播放器问题

**Q8：播放器打不开 / 从登录就卡住 / 之前能开今天不行**
1. **检查网络**（校园网限制、杀软拦截）。
2. **彻底退出进程**（任务管理器结束）后重开。
3. **重新登录**；若登不上可能是服务器端问题，稍后再试。
4. 持续不行：从打开程序到卡住录一段视频私信助教。

**Q9：播放器每次都要重装 / 解压后怎么放？**
- 播放器**只需装一次**，无需每次重装。打开直播链接失败时：确认压缩包正确解压到桌面、文件夹没被移动删除；清浏览器缓存或换浏览器；必要时重新解压原始压缩包。
- 收到新压缩包：放桌面解压到当前文件夹，旧版播放器文件可删除、新版自动生效；删旧文件后仍打不开就检查是否解压完全或重启电脑。

**Q10：播放器/输入框按回车没反应 / 显示乱码**
- **先确认光标在输入框内**，否则按回车无响应。
- 关闭窗口重开重试；检查是否最新版或换兼容设备。
- 乱码多为编码/字体问题，可换设备或反馈助教。

### 11.4 订阅与支付（Claude / ChatGPT）

**Q11：怎么订阅 Claude / ChatGPT Plus？能用礼品卡、PayPal、Apple Pay 吗？**
- 订阅步骤本身不难，**难点在国外银行卡**。官方要求**国外银行发行的信用卡/借记卡**。
- 网上流传的替代（淘宝代充、苹果礼品卡、PayPal、Apple Pay）**均有封号风险**——PayPal 最终也要求国外卡，Apple Pay 也有被封案例。
- 最稳：用自己或朋友的**真实国外银行卡**（直接绑卡，避免第三方支付渠道）。
- **强烈建议国内学员改用 DeepSeek（cc-switch 接入），充约 100 元即可，效果足够课程使用**，无需国外卡、无封号风险。

**Q12：同一张国外卡能开几个账号？**
两三个一般问题不大；但**尽量直接绑卡、避免 Apple Pay 等第三方支付**（有封号案例），且各账号注册信息真实、不重复触发安全审查。

**Q13：怎么验证 ChatGPT 账号/API 开通成功？**
- **Plus 订阅**：登录 chat.openai.com，看右上角是否显示 "ChatGPT Plus"，或在模型选择里能否切到 GPT-4。
- **API**：登录 platform.openai.com 看 Usage 是否有额度，或直接调用一次 API 看是否返回；到 Billing 页确认支付状态。

**Q14：注册 Claude 提示 "Claude is not available to new users right now"？**
这是 Anthropic 阶段性限制新用户注册。**直接走 DeepSeek + cc-switch 路线**最省事；想要官方账号可稍后再试或换网络环境。

**Q15：Skills 课前要不要先弄？需要科学上网吗？**（★）
- Skills 多在课堂中讲解，**课前无需专门操作**，先把基础环境配好即可。
- 从 GitHub 装 Skills/Plugins 需科学上网；离线 ZIP 安装则不需要。

> **AI 辅助提示词（课程后勤类问题）**：课程后勤（资料/回放/设备/支付）类问题，AI 给的多是通用建议，**最终请以群公告和助教答复为准**。可用此模板请 AI 起草私信：「帮我写一条给助教的简短私信：我是 [姓名]，预留邮箱 [xxx]，遇到 [没收到资料 / 想换设备看回放 / 播放器打不开]，请协助处理。」

**补充两条高级报错说明：**

- **Stata MCP / 长任务报 `timeout=600s`**：这是 Claude Code 内置安全机制——任务超过 600 秒（10 分钟）会被判"挂起"切断。任务开始前告诉 AI：「如果需要长时间等待，请使用 `run_in_background: true` 模式，不要阻塞当前会话。」即可后台运行不被中断。
- **Web Search 报错**：用 `claude --dangerously-skip-permissions` 启动后再跑 web search（跳过权限限制）。日常优先用 `--permission-mode auto`，仅在明确需要时临时用危险模式。临时放宽权限也可按 <kbd>Shift</kbd>+<kbd>Tab</kbd> 切到 Auto Mode 避免频繁确认。
- **Homebrew 报 `curl: (56) Recv failure` / `git fetch failed, retrying`**：多为网络波动临时失败，通常自动重试可忽略；反复失败就挂梯子/换国内镜像，或把 Homebrew 仓库改为 HTTPS 协议（见 [1.6](#16-homebrewmacos-专属)）。

## 附录 A：文档没覆盖的问题，如何借助 AI 一步步自助排查（含提问模板）

> 本课程的核心理念之一就是：**遇到报错，先问 AI。** 装环境、跑命令、改代码的问题，90% 可以靠下面这套方法自己解决。

### A.1 自助排查四步法

1. **完整复制报错**：把终端里**从命令到报错的整段文本**（或截图）复制下来——这比只说"报错了"有效十倍。
2. **套用提问模板**（见 A.2）发给 AI：Claude Code / OpenCode 网页或终端、或 **DeepSeek 网页版**（聊两个来回最快）。
3. **照着 AI 给的命令一步步做**，每步把新的输出再发回去，让它根据结果继续指导（这就是"A 修复 B"——用一个能跑的工具去修另一个出问题的工具）。
4. **仍解决不了**：精简成一句话（环境+做什么+报什么错+已试什么）发到群里或私信助教。

### A.2 万能提问模板（复制即用）

```
我在学习连享会 2026 AI 专题课程时遇到了问题，请帮我逐步排查解决。

【我的环境】
- 操作系统：Windows 11 / macOS（版本）
- 已装工具：Node.js v__、Python 3.__、Claude Code v__、cc-switch v__
- 使用的 AI 工具 / 模型：Claude Code + DeepSeek（deepseek-v4-pro）/ OpenCode

【我想做什么】
（一句话说明目标）

【实际发生了什么 / 报错信息】
（把终端从命令到报错的整段原文粘贴在这里，不要只写"报错了"）

【我已经尝试过】
（列出你试过的方法和结果）

请：①判断这是网络、权限、版本兼容还是配置问题；②给出可直接复制运行的命令，分步骤；③如果某步可能再报错，提前说明对应处理；④如果有更省事的替代方案（比如改用 cc-switch + DeepSeek 绕过），也请告诉我。
```

### A.3 让 AI 帮你"写提示词"

如果你连"该怎么问"都不确定，可以先让 AI 帮你把问题问清楚：

```
我想用 Claude Code 完成这个任务：[用大白话描述你的目标]。
我是科研新手，不确定怎么写提示词最有效。
请帮我：①把它改写成一段结构清晰、可直接发给 Claude Code 的提示词（含目标、要求、输出格式、结束标准）；②指出我还需要补充哪些信息才能让 AI 做得更好。
```

### A.4 几条让 AI 更靠谱的通用经验

- **给标准和边界**：AI 是放大器，告诉它"什么是好"（举正反例、给样例），别指望它自己猜。
- **分步而非一口吃成**：复杂任务用 `/do-agent` 或显式让它"先规划再执行、逐项打勾"。
- **PDF 先转 MD** 再让它读，引用更准、幻觉更少。
- **它偷懒/瞎说时**：加强语气、要求逐条核对、让它引用原文出处。
- **上下文快满了**：用 `/compact` 压缩保留计划，再继续。

**安装类**
- 通过镜像下载的 Claude Code 版本是否偏低、是否按推荐设定即可？→ 见 [2.2](#22-登录与模型接入重点)，先保证能跑通。
- 装的 CC 在 VS Code 里没有 `/plugin` 只有 `/manage plugin`，提示是 Kiro 第三方代理 → 多为非官方 CC 分发版；建议卸载后用 [0.3 推荐路线](#03-国内学员推荐路线无需稳定梯子) 重装官方 CLI。
- 怎么确保自己装的与老师 PDF 里功能一致？→ 用 [0.4 自检清单](#04-如何确认环境配置成功通用自检清单) 逐项核对。

**网络/API 类**
- 换了几个代理仍是同样问题 → 见 [第五章](#第五章--网络与代理科学上网设置)，优先改走 cc-switch + DeepSeek。
- 之前正常今天连不上 API → 见 [Q6](#第五章--网络与代理科学上网设置)：重启网络、cc-switch 重配、确保单一代理。
- 智谱已付费仍提示余额不足 → 见 [4.2 Q7](#42-模型-apias概念配额与费用)。

**模型/工具选择类**
- 有没有必要接 DS 模型，CC 原生模型已够强？→ 国内直连官方模型受限且易封号，接 DeepSeek 更稳；详见 [第八章](#第八章--模型选择与对比)。
- 上课需要用秘塔搜索吗？→ 非必需，能联网检索的方案（Tavily MCP / 模型自带）即可，以课堂演示为准。

**其他个人化现象**（建议直接发 AI 排查）
- "另一个 prompt 运行了，回到 PowerShell 显示这个正确吗？"
- "VS Code 打开后导入的文件夹突然不显示了，模型还变成 copilot？"
- "Node.js finish 后自动装很多东西要等吗？" → 见 [1.2 Q1](#12-nodejs-与-npm)。
- "Mac 播放软件无法验证" → 多为 Gatekeeper 拦截：系统设置 → 隐私与安全性 → 仍要打开；或右键 → 打开。
- "录制视频申请访问需要审批通过吗？" → 以助教/平台答复为准。

> 对以上任何一条，标准做法都是：**把现象 + 截图套进附录 A.2 模板发给 AI，跟着它一步步做。**
